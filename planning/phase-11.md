## Phase 11 — Animation and Interactions

### Goals

- GSAP-driven interactions that guide attention and confirm feedback
- Purposeful motion, not generic
- Handle reduced-motion and edge cases

### Checklist

- [ ] Key components have purposeful hover/focus interactions
- [ ] GSAP timelines for complex sequences; CSS for simple transitions
- [ ] Favor transform/opacity; avoid layout thrash
- [ ] Reduced-motion fallbacks and touch-device handling

### Deliverables

- Animation goals and triggers per component/page
- Motion spec (timing, easing, sequence)
- Implementation notes (GSAP vs CSS)
