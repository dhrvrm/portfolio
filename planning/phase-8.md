## Phase 8 — Page Transitions

### Goals

- Header stays fixed during route changes
- Creative transition overlay shows destination before content swap
- Smooth, fast, purposeful (not generic entrance animation)

### Checklist

- [ ] Header remains visible and stable on navigation
- [ ] Transition div/overlay appears with destination context (e.g. page name, visual cue)
- [ ] Animation is smooth, fast, and creative
- [ ] Respect reduced-motion preferences

### Deliverables

- Transition spec (timing, easing, sequence)
- Implementation notes for Astro ViewTransitions or custom solution
